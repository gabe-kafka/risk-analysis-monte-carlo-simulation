    01 · THE SETUP
    
      
    The project is a directed acyclic graph of n activities with finish-to-start, zero-lag precedence. Each activity has a baseline duration di and a set of predecessors P(i).
    
      
    02 · THE FORWARD PASS
    
      
    Given a topological ordering, CPM computes early start (ES) and early finish (EF) recursively:
    
      
    ES_i = max(EF_j : j ∈ P(i))
EF_i = ES_i + d_i
T_det = max(EF_i)
    
      
    03 · WHY DETERMINISTIC ISN'T ENOUGH
    
      
    The baseline tells us what would happen if every activity took exactly its estimated duration. But durations are uncertain — weather, labor, permits, groundwater, design changes. We need a probabilistic view.

    01 · THREE-POINT ESTIMATES
    
      
    Each activity gets three multiplicative factors from the risk register — optimistic, most-likely, pessimistic:
    
      
    (a_i, m_i, b_i) = (α·d_i, μ·d_i, β·d_i)
    
      
    02 · THE TRIANGULAR DENSITY
    
      
    Activity duration D_i is modeled as a triangular random variable:
    
      
    D_i ~ Tri(a_i, m_i, b_i)

         2(x - a)
f(x) = ──────────       a ≤ x ≤ m
       (b-a)(m-a)

         2(b - x)
f(x) = ──────────       m < x ≤ b
       (b-a)(b-m)
    
      
    03 · RISK REGISTER → DISTRIBUTION
    
      
    Identified risks are encoded into the distribution width: activities exposed to multiple risks get wider pessimistic tails (higher β). No discrete risk events are modeled separately.
    
      
    e.g. A402 Pile Installation: α=0.60 β=2.20 (5 risks)
